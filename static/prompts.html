Set up Firestore security rules

Run in Firebase Console → Firestore → Rules.

Write Firestore security rules for ClawMarketer where: - Users can only read/write their own data under users/{userId}/reports - No public access to any data - The collection structure is: users/{userId}/reports/{reportId} Give me the rules to paste into the Firebase Console.

Add a new metric to the analysis

Add "Frequency" as a metric to ClawMarketer's analyzer. Frequency = Impressions / Reach. High frequency (above 3.0) means users are seeing the same ad too many times — add this as an anomaly flag. Files to modify: agents/analyzer.py Show the anomaly as: "High frequency ({value}x) on campaign: {name}"

Add ad-set level breakdown

Add an "Ad Sets" tab to the ClawMarketer results panel that shows ad-set level performance data. The Meta API fetch already supports level="adset" (see agents/fetcher.py). Add a new API endpoint /api/run-adsets and a fourth tab in the results panel showing the same table as campaigns but for ad sets.
